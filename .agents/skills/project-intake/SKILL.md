# Project Intake

Use this skill when starting a new Codex-managed project or preparing a project-specific harness.

## Required Inputs

- Business purpose
- Target users or stakeholders
- Expected deliverables
- Known constraints
- Timeline or urgency, if relevant

## Workflow

1. Summarize the business purpose in plain language.
2. Define in-scope and out-of-scope work.
3. Identify stakeholders, systems, and data sources.
4. Capture success criteria and acceptance signals.
5. Flag sensitive data, secrets, or compliance concerns.
6. Write the intake output as a handoff-ready markdown summary.

## Expected Output

- Project purpose
- Scope boundaries
- Stakeholder and system notes
- Success criteria
- Risks and open questions

## Validation

- The purpose explains why the project exists.
- Scope boundaries are concrete enough to guide design.
- Sensitive data handling is explicitly noted.
