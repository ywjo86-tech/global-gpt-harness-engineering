# Technical Docs Harness

Use this skill when creating or improving developer-facing documentation for APIs, services, libraries, tools, or workflows.

## Inputs

- Code, API spec, or endpoint list
- Target audience
- Documentation goal
- Existing docs or examples
- Verification method

## Agent Team Pattern

- Docs lead: defines structure and audience fit.
- Endpoint analyst: extracts behavior, parameters, responses, and errors.
- Explanation writer: writes conceptual and procedural docs.
- Example builder: creates realistic usage examples.
- Completeness reviewer: checks accuracy, coverage, and usability.

## Workflow

1. Identify audience, use cases, and documentation scope.
2. Analyze endpoints, commands, or public interfaces.
3. Draft explanations for concepts and workflows.
4. Generate examples with inputs, outputs, and error cases.
5. Review docs against source behavior.
6. Check for missing prerequisites, edge cases, and unclear wording.
7. Produce final docs and a verification summary.

## Outputs

- Documentation outline
- Endpoint or interface notes
- Draft documentation
- Usage examples
- Completeness review
- Follow-up gaps

## Validation

- Examples match actual interfaces.
- Required parameters and errors are documented.
- Docs are usable by the target audience.
- Claims are traceable to code, specs, or verified behavior.
