# Request Intent Definition Harness

Use this skill when a user request is broad, ambiguous, high-impact, or likely to hide unstated business intent.

## Inputs

- User's surface request
- Available context
- Target audience or stakeholder
- Known constraints
- Desired decision or output

## Agent Team Pattern

- Intent lead: identifies the actual purpose behind the request.
- Requirements analyst: turns intent into requirements.
- Constraint analyst: identifies limits, risks, and non-goals.
- Ambiguity reviewer: separates blockers from safe assumptions.
- Deliverable framer: defines final output and acceptance criteria.

## Workflow

1. Restate the surface request.
2. Infer and validate the likely real purpose.
3. Identify functional requirements and non-functional expectations.
4. Capture constraints, risks, non-goals, and sensitive data concerns.
5. List ambiguities and classify them as blockers or assumptions.
6. Define the final deliverable, format, and acceptance criteria.
7. Recommend the next action or execution harness.

## Outputs

- Surface request summary
- Real purpose statement
- Requirements and constraints
- Ambiguity list
- Assumptions
- Final deliverable definition
- Recommended next step

## Validation

- The purpose explains why the user needs the work.
- Ambiguities are not hidden inside assumptions.
- Final deliverable is concrete enough to execute.
- High-risk unknowns are escalated before implementation.
