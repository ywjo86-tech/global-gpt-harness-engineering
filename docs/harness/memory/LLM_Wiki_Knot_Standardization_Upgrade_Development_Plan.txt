6. AGENTS.md UPDATE

Add the following policy section to AGENTS.md:

## Memory and LLM Wiki Policy

- Do not rely only on chat history for long-term project memory.
- Important decisions, project states, errors, fixes, and handoff notes must be summarized into Markdown wiki pages.
- Raw inputs must be stored in inbox before being converted into curated wiki pages.
- Use Obsidian-compatible Markdown links: [[Page Name]].
- Before creating a new page, check whether a related page already exists.
- Do not overwrite historical facts. Move outdated facts into History or Past Issues sections.
- Current Status must represent the latest confirmed state.
- Unverified assumptions must be marked as Open Questions.
- Run knot health checks after large memory updates.
- Jarvis Assistant main brain integration must remain disabled until the dedicated main brain upgrade phase.
