# Feedback Loop Standard

## Purpose

This standard defines how to turn AI mistakes into harness improvements instead of one-time fixes.

## Standard Flow

1. 오류 발견
2. 오류 유형 분류
3. 원인 확인
4. 재발 방지 규칙 작성
5. 운영지침/스킬/체크리스트 중 반영 위치 결정
6. 문서 업데이트
7. 다음 작업에서 검증

## What to Record

- What failed
- Why it failed
- Which document must change
- How the next run will verify the fix

## Example

Problem:
AI answered with outdated information.

Cause:
No freshness-check rule existed for current information.

System Improvement:
Add a rule that current or time-sensitive information must be verified with an up-to-date source.

Apply To:
- global operating principles
- research/reporting skill
- validation checklist

## Key Message

> "A repeated AI mistake is not only a bad answer. It is a signal that the harness needs improvement."

실수는 한 번 수정하고 끝내는 대상이 아닙니다.  
같은 실수가 반복되면, 그건 하네스가 더 좋아져야 한다는 신호입니다.

## Practical Rule

같은 오류가 다시 나오면, 답변만 고치지 말고 문서와 검증 절차를 함께 고치세요.
