# Prompt Engineering vs Harness Engineering

## Purpose

This guide explains the difference in a way that is easy for newcomers to understand.

## Comparison

| Topic | Prompt Engineering | Harness Engineering |
| --- | --- | --- |
| Goal | Better answer now | Reliable repeated workflow |
| Main tool | Better instruction | Better environment |
| Memory | Repeated in chat | Stored in docs and skills |
| Quality control | Manual review | Checklist and QA gate |
| User role | Asker | Operator / system designer |
| Failure handling | Ask again | Improve the system |

## Simple Interpretation

- 프롬프트 엔지니어링은 한 번의 대답을 더 좋게 만듭니다.
- 하네스 엔지니어링은 반복 업무 전체가 안정적으로 돌아가게 만듭니다.
- 프롬프트는 말의 품질을 높이고, 하네스는 일하는 구조를 높입니다.

## When to Use Each

- 빠른 답변이 필요하면 프롬프트를 다듬습니다.
- 같은 업무를 계속 반복해야 하면 하네스를 설계합니다.
- 실수가 반복되면 프롬프트만 바꾸지 말고 운영 구조를 바꿉니다.

## Conclusion

> "Prompt engineering improves a single interaction. Harness engineering improves the operating system around repeated AI work."
