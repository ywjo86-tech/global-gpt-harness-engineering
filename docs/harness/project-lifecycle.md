# Project Lifecycle

## Standard Phases

1. Intake
2. Requirements analysis
3. Harness and agent-team design
4. Solution design
5. Implementation planning
6. Development
7. Testing
8. Documentation
9. Release review
10. Deployment handoff

## Phase Outputs

- Intake: purpose, scope, stakeholders, success criteria.
- Requirements analysis: requirements, constraints, risks, acceptance criteria.
- Harness design: agent roles, skills, docs, templates, handoff files.
- Solution design: architecture, data flow, operational assumptions.
- Implementation planning: target files, steps, verification method.
- Development: implementation changes and local validation.
- Testing: automated or manual verification evidence.
- Documentation: user, operator, or developer-facing docs.
- Release review: QA findings and readiness decision.
- Deployment handoff: final operating notes and ownership.

## Control Points

Before editing files, identify the plan and target files. Before release, verify security, docs, tests, and GitHub readiness.
