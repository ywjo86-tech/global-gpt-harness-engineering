# Stage Gate Final Report Template

## Purpose

Use this one-page template to report a completed stage gate decision after fan-out and fan-in are finished.
Keep the report read-only, evidence-based, and tied to a single phase boundary.

## Report Header

- Project:
- Phase:
- Reviewer:
- Date:
- Decision: `GO` / `CONDITIONAL GO` / `NO-GO`

## Completion Summary

- Completion criteria checked:
- Evidence reviewed:
- Fan-in reviewed:
- Open questions:
- Remaining risks:

## Decision Details

### GO

- Authorization: next phase may start
- Next step:

### CONDITIONAL GO

- Conditions:
- Authorization: next phase may start after conditions are satisfied
- Next step:

### NO-GO

- Blocker summary:
- Next step:

## Files and Validation

- Changed files:
- Validation method or commands:
- Validation results:
- Scope exceptions:

## Orchestration Notes

- Fan-out threads used:
- Fan-in decision:
- User intervention required:
- Why user input is needed:
- Phase remaining paused:
- Approval boundary:
- Separate user approval required:

## Handoff Notes

- Final handoff readiness:
- Follow-up items:
- Owner of next action:

## Standard Record Shape Reminder

Use the same core fields across projects:

- `decision`
- `phase`
- `completion_criteria_checked`
- `evidence_reviewed`
- `fan_in_reviewed`
- `open_questions`
- `remaining_risks`
- `next_step`
- `authorization` or `conditions` or `blocker_summary`
- approval boundary and separate user approval requirement, when relevant
