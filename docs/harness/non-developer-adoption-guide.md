# Non-Developer Adoption Guide

## Purpose

This guide shows non-developers how to start using Harness Engineering on one repeatable task.

## Start Here

1. 반복 업무 하나를 선택한다.
2. AI 업무 폴더를 만든다.
3. 운영지침서를 작성한다.
4. 반복 업무를 스킬 문서로 나눈다.
5. 결과물 검증 체크리스트를 만든다.
6. 실수 기록을 남긴다.
7. 실수를 운영지침/스킬/체크리스트에 반영한다.
8. 작게 성공한 후 다른 업무로 확장한다.

## Recommended Folder Structure

```text
AI_Workspace/
  00_operating_rules/
  01_report_templates/
  02_email_templates/
  03_checklists/
  04_reusable_skills/
  05_error_log/
  06_outputs/
```

## What Goes Where

- `00_operating_rules/`: 언제, 어떻게 일을 맡길지 정하는 기본 규칙
- `01_report_templates/`: 보고서나 정리문 형식
- `02_email_templates/`: 자주 쓰는 이메일 형식
- `03_checklists/`: 결과물을 확인하는 점검표
- `04_reusable_skills/`: 반복 업무를 잘하는 방법을 정리한 문서
- `05_error_log/`: 실패, 실수, 수정 내용을 기록하는 곳
- `06_outputs/`: 실제로 생성된 결과물 저장소

## First Project Rule

처음부터 많은 업무를 묶지 마세요.  
하나의 반복 업무를 성공시키고, 그 방식이 안정적일 때만 확장하세요.

## Example

예를 들어 주간 보고서를 반복해서 만든다면:

- 운영지침서에 보고 시점과 입력 자료를 정합니다.
- 스킬 문서에 작성 순서와 문장 규칙을 정리합니다.
- 체크리스트에 누락 항목과 검토 항목을 넣습니다.
- 실수 기록에 반복되는 오류를 남깁니다.
