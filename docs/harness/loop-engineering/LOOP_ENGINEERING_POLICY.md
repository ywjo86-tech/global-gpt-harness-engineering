# Loop Engineering 정책

## 목적

이 저장소는 코드 작업 운영 정책으로 Loop Engineering을 사용한다.
전역 스캐폴드는 `.codex-loop-global/`에 있다.
이 정책은 큰 범위의 수정, 반복 실패, 검증 없는 합치기를 줄이고 가장 잘 검증된 방식을 보존하기 위해 존재한다.

## 운영 모델

- 전역 정책이 기준을 정한다.
- 프로젝트 정책은 전역 정책을 상속한다.
- 서브에이전트는 지정된 범위 안에서만 실행한다.
- 메인 에이전트가 최종 상태, 합치기 적합성, QA 결과, 기록 관리를 책임진다.

## 필수 루프

1. 평가
2. 제안
3. 검증
4. 합치기
5. QA 평가
6. 기록

## 핵심 규칙

- 한 루프에는 한 가지만 바꾼다.
- 수정 전에 평가 기준을 고정한다.
- 실패한 루프를 기록한다.
- 실패한 접근 방식을 반복하지 않는다.
- 검증 전에는 합치지 않는다.
- 모든 루프를 기록한다.
- 더 좋은 성공 루프가 증명되면 기존 기록은 `Superseded`로 남긴다.

## 증거 규칙

- 정량 검증을 우선한다.
- 빌드, 테스트, 린트, 타입 체크, QA 결과를 증거로 사용한다.
- QA 실패는 완료가 아니다.

## 경계 규칙

- 서브에이전트는 원본 정책을 바꿀 수 없다.
- 서브에이전트는 역할 경계를 지켜야 한다.
- 프로젝트 수준 실행은 이 정책을 상속한다.
