# V02 Operation Guide

This guide defines the stable operating rules for Global Agent Feature Development V02.

## Reading Order

1. `PLAN_INDEX.md`
2. `PLAN_STATE.md`
3. `MAIN_DEVELOPMENT_PLAN.md`
4. The active detail plan
5. `STAGE_EXIT_STANDARD.md`

## Core Rules

- Use a hierarchical plan set with main, detail, nested, and handoff documents.
- Keep each loop to one target.
- Treat the main development plan as the control document.
- Use the active detail plan for the current loop.
- Create nested plans only when a step is too complex.
- Record scope candidates instead of expanding scope silently.
- Keep UI, PC control, and approval handling aligned with the existing harness rules.

## State Rule

- Use `PLAN_STATE.md` for current phase, active routing, and progress state.
- Update `PLAN_STATE.md` when the active routing changes.

## Exit Rule

- Use `STAGE_EXIT_STANDARD.md` for the stage-end review and handoff format.
